# 2주차 과제: 프롬프트 엔지니어링 기법 조사 및 실습

## 1. 2주차 이론 과제: 프롬프트 엔지니어링 기법 조사

### 1.1. 프롬프트 기법 4가지 정리

**1. Zero-shot Prompting**
* **한 줄 정의:** 모델에게 정답 예시를 제공하지 않고, 지시사항만으로 작업을 수행하게 하는 기법입니다.
* **핵심 원리:** 방대한 데이터로 사전 학습된 모델의 기존 지식과 추론 능력을 활용하여, 처음 보는 작업에 대해서도 다음 단어를 예측하여 답변을 생성합니다.
* **사용 예시:**
```
이미지에서 질문에 맞는 답변을 추출하여 제시하라.
```
* **언제 쓰면 좋은지 / 언제 효과가 적은지:** 일반적이고 단순한 작업에 유리하며 빠르고 비용이 적게 듭니다. 하지만 복잡한 도메인 지식이 필요하거나 출력 형식을 엄격하게 맞춰야 하는 작업에서는 환각이 발생할 확률이 높고 효과가 적습니다.

**2. Few-shot Prompting**
* **한 줄 정의:** 모델에게 지시사항과 함께 2~5개 정도의 정답 예시를 제공하여 문맥을 파악하게 하는 기법입니다.
* **핵심 원리:** 문맥 내 학습(In-Context Learning)을 통해 모델이 예시의 패턴과 출력 형식을 파악하여, 사용자가 원하는 특정한 형태의 텍스트를 생성하도록 유도합니다.
* **사용 예시:**
```
Q: 65세 이상 1종 수급권자가 틀니 시술을 받았고 시술비가 1,200,000원입니다. 본인부담금은 얼마인가요?
A: 60,000원

Q: 65세 이상 1종 수급권자가 틀니 시술을 받았고 시술비가 1,200,000원입니다. 본인부담률은 얼마인가요?:
A: 5%

Q: 2종 수급권자인 생후 8개월 아기가 만성질환자이고 제2차의료급여기관에서 외래 진료를 받으면 본인부담률은 몇 %인가요?:
A: 무료

Q: 2종 수급권자인 임산부가 자연분만으로 제2차의료급여기관에서 외래 진료를 받으면 본인부담률은 몇 %인가요?:
A:  
```
* **언제 쓰면 좋은지 / 언제 효과가 적은지:** 출력 형식을 고정해야 하거나 Zero-shot으로 해결되지 않는 작업에 효과적입니다. 그러나 다단계 논리적 추론이 필요한 복잡한 문제에서는 단순 예시 제공만으로는 한계가 있습니다.

**3. Chain-of-Thought (CoT)**
* **한 줄 정의:** 모델이 최종 정답을 내리기 전에, 중간 추론 과정(사고의 흐름)을 명시적으로 작성하도록 유도하는 기법입니다.
* **핵심 원리:** 복잡한 문제를 여러 개의 작은 단계로 나누어 처리하게 함으로써, 모델이 각 단계의 중간 결과를 바탕으로 다음 논리를 전개하여 최종 답변의 정확도를 높입니다.
* **사용 예시:**
```
Q: 65세 이상 1종 수급권자가 틀니 시술을 받았고 시술비가 1,200,000원입니다. 본인부담금은 얼마인가요?:
A: 수급권자 구분: 1종(65세 이상), 진료 구분: 틀니, 발생 금액: 1,200,000원 => 본인부담금이므로 1,200,000원 * 5% = 60,000원이다.

Q: 65세 이상 1종 수급권자가 틀니 시술을 받았고 시술비가 1,200,000원입니다. 본인부담률은 얼마인가요?:
A: 수급권자 구분: 1종(65세 이상), 진료 구분: 틀니, 발생 금액: 1,200,000원 => 본인부담률이므로 5%

Q: 2종 수급권자인 생후 8개월 아기가 만성질환자이고 제2차의료급여기관에서 외래 진료를 받으면 본인부담률은 몇 %인가요?:
A: 수급권자 구분: 2종(6세 미만), 진료 구분: 외래 진료, 발생 금액: 아직 발생하지 않음 => 무료 

Q: 2종 수급권자인 임산부가 자연분만으로 제2차의료급여기관에서 외래 진료를 받으면 본인부담률은 몇 %인가요?:
A:  
```
* **언제 쓰면 좋은지 / 언제 효과가 적은지:** 수학 계산, 상식 추론, 복잡한 논리 문제에서 성능이 크게 향상됩니다. 반면, 매우 단순한 정보 추출 작업에서는 불필요한 토큰 낭비와 시간 지연을 초래합니다.

**4. Self-Consistency**
* **한 줄 정의:** 동일한 프롬프트를 여러 번 반복 실행하여 다양한 추론 경로를 생성한 뒤, 가장 많이 나온 답변(다수결)을 최종 정답으로 채택하는 기법입니다.
* **핵심 원리:** 단일 추론 경로에서 발생할 수 있는 모델의 무작위한 오류를 다수결 투표라는 통계적 방법을 통해 상쇄하여 결과의 신뢰성을 극대화합니다.
* **사용 예시:** 
```
Q: 65세 이상 1종 수급권자가 틀니 시술을 받았고 시술비가 1,200,000원입니다. 본인부담률은 얼마인가요?
A: 의료급여 규정에 따라 65세 이상 1종 수급권자의 틀니 시술 본인부담률은 5%로 정해져 있습니다. 정답은 5%입니다.

Q: 65세 이상 1종 수급권자가 틀니 시술을 받았고 시술비가 1,200,000원입니다. 본인부담금은 얼마인가요?
A: 1종 수급권자의 틀니 시술 본인부담률은 5%입니다. 시술비 1,200,000원의 5%를 계산하면 1,200,000 * 0.05 = 60,000원입니다. 정답은 60,000원입니다.

Q: 2종 수급권자인 생후 8개월 아기가 만성질환자이고 제2차의료급여기관에서 외래 진료를 받으면 본인부담률은 몇 %인가요?
A: 15세 이하 아동 중 1세 미만 만성질환자가 2차 의료급여기관에서 외래 진료를 받을 경우 본인부담금은 전액 면제됩니다. 정답은 무료입니다.

Q: 2종 수급권자인 임산부가 자연분만으로 제2차의료급여기관에서 외래 진료를 받으면 본인부담률은 몇 %인가요?
A:
```
* **언제 쓰면 좋은지 / 언제 효과가 적은지:** 높은 정확도와 신뢰성이 요구되는 복잡한 작업에 매우 효과적입니다. 하지만 API 호출 횟수와 토큰 사용량이 배수로 증가하므로, 비용과 처리 속도가 중요한 서비스에는 부적합합니다.

## 3. 실습 과제 예측 및 결과 비교

### 2.1. Zero-shot 정답률 예측
* **가설:** 50~60% 수준의 정답률을 기록할 것으로 예측했습니다. 프롬프트에 본인부담률을 실제 금액(원)으로 환산하라는 명시적인 명령을 포함하지 않았기 때문에, Hard 난이도의 문항들에서 금액 대신 '00%' 형태의 비율로 답변하여 오답 처리될 것이라 예상했습니다.
* **실제 결과:** Zero-shot 환경에서 96.67%(30문제 중 29문제 정답)라는 높은 정답률을 기록했습니다.
* **결과 분석:** 별도의 예시 없이도 질문의 맥락과 제공된 진료비(시술비) 데이터를 스스로 조합하여, 별도의 지시 없이도 비율(%)을 실제 금액(원)으로 정확하게 환산해 내는 뛰어난 추론 능력을 보여주었습니다. 스키마의 description을 제거하고 다시 시도해볼 예정입니다.

### 2.2. 난이도별 개선폭 예측
* **가설:** Zero-shot 결과를 보았을 때, Hard 난이도 문항들이 예시(Few-shot)를 제공받았을 때 가장 큰 개선이 일어날 것으로 예측했습니다. 본인부담금 관련 조항이나 출력 형식에 대한 구체적인 예시를 제공하는 Few-shot 기법에서 원하는 형식으로 결과가 도출될 거라고 생각했습니다.
* **실제 결과:** Zero-shot과 Few-shot 모두 96.67%의 동일한 정답률을 기록하여, 특정 난이도에서의 극적인 개선폭은 관찰되지 않았습니다.
* **결과 분석:** 예상과 달리 별 차이가 없었기 때문에 월요일날 스키마의 description을 제거하고 다시 시도해볼 예정입니다.

### 2.3. 4가지 기법 중 가장 큰 정답률 점프 구간 예측
* **가설:** 초기에는 CoT 적용 시 가장 큰 상승이 있을 것으로 예측했으나, 프롬프트 구조를 고려했을 때 Zero-shot에서 Few-shot으로 넘어가는 구간에서 가장 큰 정답률 점프가 발생할 것으로 가설을 수정했습니다.
* **실제 결과:** Zero-shot에서 Few-shot으로 넘어갈 때 정답률 상승은 없었으며(96.67% 유지), 오히려 CoT와 Self-Consistency를 적용했을 때 정답률이 90.00%로 하락했습니다.
* **결과 분석:** 모델의 자유도(temperture)가 올라가면서 의미는 동일했으나 서술 형식이 조금 원하는 기댓값과 달라진 것으로 생각됩니다.

### 4.2 각 단계별 프롬프트 전문

#### Zero-shot Prompting
```
당신은 의료급여 전문가입니다. 첨부된 이미지를 참조하여 다음 여러 환자들의 본인부담률을 정확히 계산하세요.
```

#### Few-shot Prompting
```
당신은 의료급여 전문가입니다. 첨부된 이미지를 참조하여 다음 환자들의 본인부담률을 정확히 계산하세요.
아래의 예시를 참고하여 답변을 작성해 주세요.

[예시 1]
질문: 65세 이상 1종 수급권자가 틀니를 하면 본인부담률은 몇 %인가요?
answer: 5%
reason: 04번 표를 보면 1종 수급권자의 틀니 본인부담률은 5%로 명시되어 있습니다.

[예시 2]
질문: 15세 이하 1종 수급권자인 4세 아동이 입원하면 본인부담률은 몇 %인가요?
answer: 무료
reason: 15세 이하 아동 입원 조건에서 6세 미만은 본인부담금이 면제(무료)입니다.


```

#### Chain-of-Thought (CoT)
```
당신은 의료급여 전문가입니다. 첨부된 이미지를 참조하여 환자들의 본인부담률을 정확히 계산하세요.
단, 최종 결론을 내리기 전에 반드시 다음 단계를 거쳐 논리적으로 생각하세요(Chain of Thought):
1. 환자가 1종인지 2종인지 수급권자 유형을 먼저 파악합니다.
2. 진료받는 곳이 어떤 의료기관(1차, 2차, 3차, 약국 등)인지 확인합니다.
3. 나이, 질환(중증질환, 임산부 등) 등 예외 조건이 있는지 확인합니다.
4. 표에 맞춰 최종 본인부담률을 도출합니다.
```

#### Self-Consistency
```
당신은 의료급여 전문가입니다. 첨부된 이미지를 참조하여 환자들의 본인부담률을 정확히 계산하세요.
답을 내리기 전, 환자의 수급권자 유형, 의료기관 종류, 특이사항 등을 종합적으로 고려하여 'reasoning_process'에 사고 과정을 상세히 적으세요.
```

### 4.4 가장 어려웠던 문항 3건과 원인
기댓값과 내용은 같았으나, 답변 형식이 다르게 나온 문항들이 있었습니다.

* **q15**: 65세 이상 2종 수급권자가 임플란트를 할 때 본인부담 보상제나 상한제가 적용되나요?
    * **기댓값:** 해당되지 않음
    * **실제 출력값:** 적용되지 않음
    * 의미상으로는 동일하지만, 기댓값과 표현 형식이 일치하지 않아 오답으로 처리되었습니다.
    * 스키마에 관련 문항을 넣어 개선할 수 있을 것 같습니다.

* **q12**: 2종 수급권자의 조현병 외 정신질환 환자가 외래 진료를 받으면 본인부담률은 얼마인가요?
    * **기댓값:** 병원급 이상 10%
    * **실제 출력값:** 10%
    * CoT 및 SC 단계에서 모델이 본인부담률 수치 자체는 맞게 추출했지만, '병원급 이상'이라는 조건부 텍스트를 결과 값에서 누락하여 다르게 나왔습니다.
    * 이것도..? 스키마에 관련 문항을 넣어 개선할 수 있을 것 같습니다.

* **q04 / q14**: 본인부담금이 없는 환자 케이스
    * **기댓값:** 무료
    * **실제 출력값:** 0%
    * 비용 계산에 모델이 초점을 맞춰 '0%'라는 수치로 나온 것으로 보입니다.

#### 인사이트
* 3.1 Flash 모델은 복잡한 표 데이터에 대해서도 Zero-shot만으로 충분한 문맥 이해 및 추론이 가능함을 보였습니다.
* 오히려 CoT, Self-consistency보다 Few-shot이 더 효율적이고 충분히 원하는 정도의 추론이 가능한 것을 확인하였습니다.
* Self-Consistency 기법은 다수결을 위해 프롬프트를 다중 호출하므로, Zero-shot 대비 실행 시간은 늘어나고 비용 역시 약 3.6배 상승했습니다. 따라서 환경에 요구되는 정밀도에 맞춰 기법을 선택해야 할 것 같습니다.

### 3.6 추가 실험 계획 (영문 프롬프트 최적화)
과제가 원하는 방향으로 진행되지 않은 것 같아 휴일인 월요일날 다시 진행할 예정입니다..ㅠㅠ

이후 추가 실험으로 모든 프롬프트를 영어로 번역하여 진행할 예정입니다. LLM의 학습 데이터 대다수가 영어로 구성되어 있다는 점을 고려하여, 영문 프롬프트 환경에서 토큰 소모량, 실행 속도, 그리고 Exact Match 기준의 정답률에 어떤 유의미한 변화가 발생하는지 관찰 및 비교하고자 합니다.
