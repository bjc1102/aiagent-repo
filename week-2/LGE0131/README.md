## 1. Zero-shot Prompting (제로샷 프롬프팅)

### 정의
모델에게 어떠한 정답 예시도 주지 않고, 직접적인 지시사항과 질문만으로 답변을 유도하는 가장 기본적이고 직관적인 프롬프트 기법입니다.

### 핵심 원리 (Autoregressive 관점)
대규모 텍스트 데이터로 사전 학습된 언어 모델의 내재된 지식(World Knowledge)을 그대로 활용합니다. Autoregressive 관점에서 볼 때, 사용자가 입력한 '지시어' 텍스트 자체가 다음 토큰을 예측하기 위한 유일한 초기 문맥(Context)으로 작용하여, 모델이 학습한 분포 내에서 가장 그럴듯한 답변을 생성하도록 유도합니다.

### 사용 예시

```
- 지시: 다음 문장의 감정을 '긍정', '부정', '중립' 중 하나로 분류하시오.
- 문장: "오늘 팀원들과 함께한 프로젝트 회의는 정말 생산적이고 즐거웠어!"
- 감정: 위와 같이 분류에 대한 사전 예시 없이 질문만 던져도, 모델은 사전 학습된 언어 능력을 바탕으로 긍정이라는 토큰을 성공적으로 예측해 냅니다.
```

### 언제 쓰면 좋은지 / 언제 효과가 적은지

#### 언제 쓰면 좋은지
일반적인 상식 질문, 단순 번역, 감정 분석, 텍스트 요약 등 모델이 이미 충분히 학습한 직관적이고 보편적인 작업에 매우 빠르고 효율적입니다.

#### 언제 효과가 적은지
엄격하고 특수한 출력 형식(예: 특정 스키마의 JSON)이 필요하거나, 복잡한 다단계 논리 추론, 도메인 특화적인 문제를 해결할 때는 모델이 환각(Hallucination)을 일으키거나 엉뚱한 답을 낼 확률이 높습니다.

## 2. Few-shot Prompting (퓨샷 프롬프팅)

### 정의
원하는 답변의 형식과 논리 패턴을 모델이 학습할 수 있도록, 지시사항과 함께 소수(보통 1개~수십 개)의 '정답 예시(Demonstrations)'를 제공하는 기법입니다.

### 핵심 원리 (Autoregressive 관점)

문맥 내 학습(In-Context Learning)을 활용하는 방법입니다. Autoregressive 모델은 이전에 주어진 토큰들의 패턴을 바탕으로 다음 토큰을 예측합니다. 프롬프트에 제공된 '입력-출력' 예시 쌍들이 강력한 문맥으로 작용하여, 모델이 다음 토큰을 생성할 때 사용자가 의도한 특정 형식이나 톤앤매너로 확률 분포를 급격하게 보정(Conditioning)해 줍니다.

### 사용 예시

```
- 단어: Awesome
- 번역: 훌륭한

- 단어: Beautiful
- 번역: 아름다운

- 단어: Phenomenal
- 번역: 위와 같이 "단어-번역" 패턴의 예시를 두 개 보여준 뒤 질문을 던지면, 모델은 앞에 전개된 패턴(문맥)을 그대로 이어나가기 위해 경이로운과 같은 알맞은 번역어를 출력하게 됩니다. 새로운 사내 은어나 독특한 문체를 가르칠 때도 동일하게 적용됩니다.
```


### 언제 쓰면 좋은지 / 언제 효과가 적은지

#### 언제 쓰면 좋은지
모델이 일관된 형식의 출력을 내보내야 할 때, 또는 모델이 사전 학습하지 않은 새로운 개념/작업 스타일을 모방하게 만들 때 매우 효과적입니다.

#### 언제 효과가 적은지
예시를 많이 넣을수록 컨텍스트 길이(토큰 비용)가 증가합니다. 또한, 단순히 정답만 있는 예시를 주는 것으로는 복잡한 수학 문제나 고도의 논리가 필요한 작업의 정확도를 극적으로 올리기 어렵습니다.

## 3. Chain-of-Thought (CoT, 생각의 사슬)

### 정의
모델에게 최종 정답을 바로 요구하지 않고, 문제를 해결하기 위한 '단계별 중간 추론 과정(Step-by-step reasoning)'을 명시적으로 생성하도록 유도하는 기법입니다.

### 핵심 원리 (Autoregressive 관점)
이 기법은 Autoregressive 생성의 약점과 강점을 절묘하게 활용합니다. 복잡한 문제에서 곧바로 정답 토큰을 예측하게 하면 확률적 거리가 너무 멀어 오류가 발생합니다. 하지만 "단계별로 생각해보자"라고 지시하거나 추론 과정을 포함한 예시를 주면, 모델은 중간 논리 토큰들을 먼저 생성합니다. 자신이 방금 생성한 중간 추론 토큰들이 다시 다음 예측을 위한 '강력한 논리적 문맥'으로 작용하게 되어(징검다리 효과), 최종적으로 정확한 정답 토큰에 도달할 확률이 극대화됩니다.

### 사용 사레
일반 Few-shot에서는 질문 -> 정답 예시를 주지만, CoT에서는 풀이 과정을 추가합니다.

```
[예시] > 질문: 철수는 사과를 10개 가지고 있었습니다. 동생에게 2개를 주고, 마트에서 5개를 더 샀습니다. 지금 사과는 몇 개인가요?

풀이: 처음에 사과가 10개 있었습니다. 동생에게 2개를 주었으므로 10 - 2 = 8개가 남습니다. 마트에서 5개를 더 샀으므로 8 + 5 = 13개입니다. 따라서 정답은 13입니다.

정답: 13
```

이런 방식의 예시를 주면, 모델은 새로운 질문을 받았을 때도 예시처럼 스스로 풀이 과정을 텍스트로 풀어낸 뒤 최종 답을 도출합니다.
(Zero-shot CoT의 경우 "Let's think step by step"이라는 매직 워드를 붙이는 것만으로도 유사한 효과를 냅니다.)

### 언제 쓰면 좋은지 / 언제 효과가 적은지

#### 언제 쓰면 좋은지
수학 문제 풀이, 상식 추론, 코딩 논리 전개 등 여러 단계의 사고(Multi-step reasoning)가 필요한 복잡한 작업에서 성능을 비약적으로 끌어올릴 수 있습니다.

#### 언제 효과가 적은지
"한국의 수도는?"과 같이 단계를 나눌 필요가 없는 단순 정보성 질문이나 창의적인 아이디에이션 작업에서는 불필요한 토큰 생성으로 인해 시간과 비용만 낭비될 수 있습니다.

## 4. Self-Consistency (자기 일관성)

### 정의
CoT 기법을 사용해 여러 개의 다양한 추론 경로와 답변을 독립적으로 생성한 뒤, 그중 가장 많이 등장한 답변을 '다수결(Majority Vote)'로 최종 채택하는 앙상블 기법입니다.

### 핵심 원리 (Autoregressive 관점)
언어 모델은 디코딩 과정에서 Temperature(온도) 설정 등을 통해 매번 조금씩 다른 확률적 샘플링을 수행합니다. 복잡한 문제를 풀 때 한 번의 생성(Greedy decoding)에만 의존하면, 우연히 잘못된 토큰을 선택했을 때 추론 전체가 망가질 수 있습니다. Self-Consistency는 Autoregressive 샘플링의 다양성을 역이용하여, 여러 번의 생성을 통해 다양한 논리적 경로(다양한 토큰 시퀀스)를 탐색하고 오답 노이즈를 상쇄시켜 가장 일관되게 도출되는 '확률적 합의점'을 찾는 원리입니다.

### 사용 예시
매우 어려운 수학 문제를 모델에게 주고 동일한 프롬프트로 5번 답변을 생성하게 합니다.

```

생성 1 (CoT 적용): ...과정을 거쳐 정답: 42

생성 2 (CoT 적용): ...다른 논리를 거쳐 정답: 42

생성 3 (CoT 적용): ...중간 계산 실수로 정답: 38

생성 4 (CoT 적용): ...과정을 거쳐 정답: 42

생성 5 (CoT 적용): ...잘못된 공식 적용으로 정답: 50

```

결과적으로 5번 중 3번이 '42'를 가리키고 있으므로, 모델의 단일 환각이나 계산 실수를 배제하고 최종 정답을 42로 확정합니다.

### 언제 쓰면 좋은지 / 언제 효과가 적은지

#### 언제 쓰면 좋은지
정답이 분명히 존재하는 고난도의 수학, 논리, 상식 추론 문제에서 단일 CoT 프롬프팅만으로 한계에 부딪혔을 때, 정확도(SOTA)를 최고 수준으로 끌어올려야 하는 핵심 기능에 사용합니다.

#### 언제 효과가 적은지
동일한 요청을 여러 번(통상 5~40회) 생성해야 하므로 API 호출 비용과 응답 지연 시간(Latency)이 기하급수적으로 증가합니다. 따라서 실시간 처리가 중요하거나 텍스트 요약, 스토리텔링 등 '다수결 정답'이 없는 주관적인 개방형(Open-ended) 작업에는 부적합합니다.
