# 1. 개요

본 과제는 의료급여 본인부담률 데이터를 기반으로 LLM이 복잡한 조건을 정확하게 판단하도록 다양한 프롬프트 엔지니어링 기법을 적용하고 성능을 비교 분석하는 것을 목표로 한다.

이론적으로는 프롬프트 기법의 원리를 이해하고, 실습에서는 실제 정답률 변화를 통해 효과를 검증하였다.

# 3. 프롬프트 기법 이론 정리

## 3.1 Zero-shot Prompting

### 정의
예시 없이 지시만으로 문제를 해결하는 방식

### 원리
LLM은 autoregressive 방식으로 다음 토큰을 예측하며, 학습된 패턴을 기반으로 문제를 해결한다.

### 예시
"본인부담률을 계산하세요" → 바로 답 출력

### 장점
- 일반화 능력
- 빠른 적용

### 단점
- 정확도 낮음
- 복잡한 문제 취약

---

## 3.2 Few-shot Prompting

### 정의
입력-출력 예시를 제공하여 모델이 패턴을 학습하도록 하는 방식

### 원리
모델은 예시를 보고 동일한 패턴을 복사하는 방식으로 답을 생성한다.

### 예시
"320,000 × 0.05 = 16,000원" → 이후 문제도 동일 방식 적용

### 장점
- 가장 높은 성능 향상
- 출력 형식 안정

### 단점
- 일반화 제한
- 예시 품질 의존

---

## 3.3 Chain-of-Thought (CoT)

### 정의
문제 해결 과정을 단계적으로 추론하도록 유도하는 방식

### 원리
복잡한 문제를 조건 분해 → 규칙 적용 → 계산으로 나누어 해결

### 예시
조건 추출 → 규칙 선택 → 계산

### 장점
- 복잡한 문제 해결 가능
- reasoning 강화

### 단점
- 추론 품질 불안정
- 오히려 성능 저하 가능

---

## 3.4 Self-Consistency

### 정의
여러 번 생성 후 다수결로 최종 답 선택

### 원리
다양한 추론 경로 생성 → 가장 많이 나온 답 선택

### 장점
- 안정성 증가
- 오답 감소

### 단점
- 동일 오답 반복 시 무력
- 비용 증가

---

# 4. 실습 전 가설

### 가설 1
Zero-shot은 hard 문제에서 거의 실패할 것이다.

### 가설 2
Few-shot에서 가장 큰 성능 향상이 발생할 것이다.

### 가설 3
Self-consistency는 CoT 대비 소폭 개선될 것이다.

# 10. 최종 결론

- 최고의 조합: Few-shot + CoT + Self-consistency
- 가장 중요한 요소: 예시 품질
- CoT는 필수지만 단독 효과는 제한적
- Self-consistency는 안정성은 높지만 근본 오류 해결 못함
- 데이터 표현 방식도 성능에 큰 영향

# 11. 최종 인사이트

- LLM은 "추론"보다 "패턴 복사"에 강함
- 구조화된 데이터보다 자연어가 더 효과적일 수 있음
- Self-consistency는 다양성이 핵심
- 프롬프트보다 예시 설계가 더 중요
